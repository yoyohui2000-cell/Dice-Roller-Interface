簡單講：

**AI GM Engine（LLM） = 你的 AI 地下城主「大腦」層。**

它不是整個遊戲，也不是資料庫，而是：

> 負責「思考、裁定、扮演、敘事」的 AI 核心。

在多人網頁版裡，可以把整個系統拆成這樣：

```text
玩家(Web/手機)
        ↓
Frontend (UI介面)
        ↓
Game Server (遊戲規則/同步)
        ↓
AI GM Engine (LLM大腦)
        ↓
Memory / World DB (長期記憶)
```

每層做的事不同。

---

# 1. Frontend（玩家看到的網頁）

就是遊戲介面。

負責：

* 地圖
* 對話框
* 骰子UI
* 戰鬥介面
* 角色卡
* 聊天視窗
* 物品欄

像：

* BG3 UI
* Roll20
* Discord bot介面

它**不負責思考**。

只是顯示。

---

# 2. Game Server（遊戲伺服器）

這是「裁判＋同步器」。

多人遊戲最重要。

負責：

### 玩家同步

例如：

A 玩家在酒館
B 玩家在森林

Server知道：

誰在哪。

---

### 權威狀態（Authoritative State）

防止 AI 胡亂改。

例如：

真正 HP：

```json
{
  "goblin_01": {
    "hp": 4
  }
}
```

不是 AI 說了算。

Server 才是最終真相。

---

### 規則驗證

例如：

玩家：

> 我瞬移100公里

Server：

> 角色沒有這法術。

拒絕。

---

### 戰鬥順序

Server管理：

* initiative
* 回合
* buff/debuff
* cooldown

所以多人不會亂。

---

# 3. AI GM Engine（LLM）

這就是你問的。

它是：

**AI GM 的推理與演出核心。**

輸入：

```text
玩家行動
+
世界狀態
+
GM Prompt
+
記憶摘要
```

輸出：

```text
GM敘事
NPC行為
規則裁定建議
世界反應
```

例如：

玩家：

> 我偷偷跟蹤男爵。

Server給 LLM：

```json
玩家位置
男爵位置
夜晚
衛兵巡邏
潛行+5
世界狀態
```

AI GM Engine：

推理：

> 需要潛行檢定
> 男爵其實在密會
> 可能觸發陰謀線

然後回：

```text
[技能檢定]
潛行 DC14
```

它：

**負責判斷與敘事。**

但：

**不保存真實世界。**

---

# 4. Memory / World DB

這是：

**世界記憶庫。**

AI 本身記憶不可靠。

所以長期跑團要外接。

存：

### 玩家

```json
聲望
任務
犯罪
裝備
```

### NPC

```json
好感
位置
秘密
目標
```

### 世界

```json
戰爭
派系
時間線
事件
```

### Campaign

```json
主線
支線
伏筆
歷史
```

---

所以：

AI GM Engine ≠ 整個遊戲。

它只是：

> 「像真人 DM 一樣思考」的那一層。

---

你可以把它想成：

| 系統           | 類比     |
| ------------ | ------ |
| Frontend     | 遊戲畫面   |
| Server       | 真正裁判   |
| Memory DB    | DM筆記本  |
| AI GM Engine | 地下城主本人 |

---

這也是為什麼很多 AI RPG 做不久就崩。

因為它們其實只有：

```text
玩家
↓
LLM
↓
回答
```

沒有：

* 世界狀態
* 長期記憶
* Server權威層

所以 30 分鐘後：

NPC忘記你是誰。

如果你要做「多人線上 AI D&D」，AI GM Engine 會是整個系統最核心，但它需要 **Server + Memory** 才能變成真正可長期運作的 GM。
